# 参数说明文档

## tile_size（图片缩放分块尺寸）

### 作用
`tile_size` 参数用于控制图片缩放时的**分块处理大小**，用于降低内存占用。

### 工作原理

当需要将一张大图片缩放到智能对象的尺寸时，如果一次性处理整张图片，可能会：
- 占用大量内存
- 导致内存溢出（特别是处理超大图片时）
- 处理速度慢

**分块处理**的工作原理：
1. 将原始图片分成多个小块（每个块的大小 = `tile_size` x `tile_size` 像素）
2. 逐个处理每个小块：裁剪 → 缩放 → 粘贴到目标位置
3. 处理完一个块后立即释放内存，再处理下一个块

### 参数范围
- **最小值**: 64 像素
- **最大值**: 2048 像素
- **默认值**: 512 像素

### 使用建议

| 图片尺寸 | 推荐 tile_size | 说明 |
|---------|---------------|------|
| < 2000x2000 | 512 | 默认值，适合大多数情况 |
| 2000x2000 - 5000x5000 | 1024 | 中等大小图片 |
| > 5000x5000 | 2048 | 超大图片，需要更大分块 |

### 示例

```python
# 小图片，使用默认值
{
    "tile_size": 512  # 适合大多数情况
}

# 超大图片，使用更大的分块
{
    "tile_size": 1024  # 减少分块数量，提高处理速度
}

# 内存受限环境，使用更小的分块
{
    "tile_size": 256  # 更小的内存占用，但处理时间稍长
}
```

### 性能影响

- **tile_size 较小**（如 256）：
  - ✅ 内存占用更低
  - ❌ 处理时间稍长（分块更多）
  - ✅ 适合内存受限的环境

- **tile_size 较大**（如 1024-2048）：
  - ✅ 处理速度更快（分块更少）
  - ❌ 内存占用稍高
  - ✅ 适合处理超大图片

### verbose=false 时显示的信息

静默模式，只显示关键信息：
- 处理开始
- 处理完成
- 错误信息（如果有）

### 使用场景

**verbose=true**（推荐用于调试）：
- ✅ 开发调试时
- ✅ 排查问题时
- ✅ 需要了解详细处理过程时
- ✅ 第一次使用时

**verbose=false**（推荐用于生产环境）：
- ✅ 生产环境，减少日志输出
- ✅ 批量处理时，避免日志过多
- ✅ 集成到其他系统时，减少干扰

### 示例

```python
# 调试模式 - 显示所有信息
{
    "verbose": true
}

# 生产模式 - 静默运行
{
    "verbose": false
}
```

## 完整参数示例

```json
{
  "psd_path": "D:/templates/template.psd",
  "image_path": "D:/images/image.jpg",
  "export_dir": "D:/output",
  "smart_object_name": "图片",
  "output_filename": "result.png",
  "tile_size": 512,      // 图片缩放分块尺寸（64-2048，默认512）
  "verbose": true        // 是否显示详细信息（默认true）
}
```

## 总结

| 参数 | 作用 | 默认值 | 推荐值 |
|------|------|--------|--------|
| `tile_size` | 控制图片缩放时的分块大小，用于降低内存占用 | 512 | 512（大多数情况）或 1024（超大图片） |
| `verbose` | 控制是否显示详细的处理日志 | true | true（调试）或 false（生产） |
